Columns

Starting with bootstrap 4, css flexbox is used internally to realize the layout. The grid system is based on 12 columns. Some important points to understand the layout classes are:

  • A class with a breakpoint, applies for the given breakpoint and for all breakpoints larger than it. Below the breakpoint nothing is applied.
  • A class with a larger breakpoint overrides one that has a smaller breakpoint.
  • If no breakpoint is specified, the class applies to all viewport sizes but is overridden by any class with a breakpoint specified.
  • The row class sets the css flex-wrap property to wrap. To avoid wrapping the columns, the flex-nowrap class must be provided with the row class.

Columns helper

Use the following classes to apply the same column style to all direct children, there are two possibilities:

  1. For n=1..6 the row-cols-n class applies col-(12/n) to all direct children so that it will wrap after n-columns. Note that it works also for n=5 because it sets the column width to 20%.
  2. The row-cols-auto class applies col-auto to all direct children.

Note: the children do not need to have a column class.

Gutter

Gutters (values between 0-5) adjust the spacing between columns, not between the container and the columns.

  • gx-* classes control the horizontal gutter widths. The default is gx-4.
  • gy-* classes control the vertical gutter widths within a row when columns wrap to new lines. The default is gy-0.
  • g-* classes control both.
How it works internally
  1. For the horizontal spacing bootstrap sets left and right paddings for the columns, and negative left and right margins for the rows.
  2. For the vertical spacing bootstrap sets positive top margins for the columns and negative top margins for the rows.
  3. The bootstrap containers have left and right paddings to compensate for the negative left and right margins. Those paddings can be seen in the examples below. Check the Container padding section for more details.

Margin and Padding

Assign a margin or a padding to an element with values between 0-5 or auto. Remember that bootstrap sets box-sizing to border-box for all elements.

  • mt-* mb-* ms-* me-* classes control top, bottom, start and end margin.
  • pt-* pb-* ps-* pe-* classes control top, bottom, start and end padding.
  • mx-* px-* classes control horizontally.
  • my-* py-* classes control vertically.
  • m-* p-* classes control all.

Container padding

The bootstrap containers have left and right paddings of 12px which compensate with the row's left and right margins of -12px.

If a container has p-0 set or if a none-bootstrap container is used, then the row is dragged out of the viewport with its negative margins. This will have no real effect on the left side, but it will cause a visible overflow on the right side. If a zero padding container is needed, there are two solutions:

1. Apply overflow-hidden to the container
