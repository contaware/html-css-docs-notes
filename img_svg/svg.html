SVGs work differently from images, you cannot just do svg { width: 100%; height: auto; } to make them responsive!

Default size

A SVG without any size-related attributes will render its shapes within the SVG element with a default width of 300px and a default height of 150px.

Setting width and height

Setting the width and height attributes will change the size of the SVG element, but not scale the shapes. We can specify any length unit we want, but if we provide unitless numbers, they default to pixels. CSS width and height properties or <img>'s width and height attributes overwrite the width and height attributes of the SVG.

Setting viewBox

A viewBox is defined by x, y, width and height. The origin is the top-left corner, x grows to the right and y grows down. If a viewBox is set, that viewBox will scale with all the shapes to the given width and height according to preserveAspectRatio, which if missing, defaults to preserveAspectRatio="xMidYMid meet" and that's equivalent to <img>'s object-fit: contain. Note how the green triangle below the red one is also shown, even if it is outside the viewBox.

Setting preserveAspectRatio

In addition to preserveAspectRatio="xMidYMid meet", there is also preserveAspectRatio="xMidYMid slice" which is equivalent to <img>'s object-fit: cover. We can replace Mid with Min or Max to align the graphic flush against one side or the other. To stretch, disable aspect ratio preservation with preserveAspectRatio="none".

Carefully overwrite width and/or height to get the wished results. Do not use the auto keyword as it does nothing to SVG. Important is that the SVG file has a viewBox attribute and to stretch preserveAspectRatio="none" is necessary.
