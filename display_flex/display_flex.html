Flexbox concepts

  • The flexible box layout is a one-dimensional model which determines the Free space and grows/shrinks/distributes the flex items in that space.
  • Free space = Flex container size - Total flex-basis size of all flex items - Total gaps
  • The Free space can be positive or negative.
  • The main axis is the direction in which the flex items are placed in the flex container. The other axis, which is perpendicular to the main axis, is called the cross axis. The main axis is set by flex-direction which defaults to row, the other value is column.

Flexbox flex

  • flex-basis specifies the initial size of the flex items before the free space is distributed according to the flex factors. Default for flex-basis is auto which means that the item's width/height property is used if it is set, otherwise the content size is taken. Setting flex-basis to content means that only the content size is employed, the item's width/height propriety is ignored.
  • A flex-basis set to a px or a % value, overrides the item's width/height property. The percentage is relative to the size of the flex container. If an item has borders or a padding, its box-sizing property should be set to border-box. This will prevent overflowing the flex container.
  • Default for flex-grow is 0 (no grow), positive values mean grow in the available positive free space with the given proportion.
  • Default for flex-shrink is 1 (shrink), positive values mean reduce the negative free space with the given proportion.
  • An item's min-width/min-height or a max-width/max-height will always limit the flexing.
  • The gap property explicitly controls the space between flex items. It applies that spacing only between items not on the outer edges.
  • Shorthand flex: flex-grow flex-shrink flex-basis;

Flexbox items order

Items with the same order value belong to the same ordering group. The default order value is 0. Items are placed in visual order according to their order value, lowest values first. If more than one item has the same order value, then within that group the items are laid out as per html order.
