• The default for height is auto, which means that a element sets its height so that it can contain the children.
  • As a parent wants to be the smallest height required to contain its children, to avoid a circular paradox, we have to give the parent a fixed height when the children use percentage heights.
  • The <html> tag has no parent, and if we set height: 100% for it, the viewport height is used.
    The following works well: html, body { height: 100%; }.
  • The height property with percent is calculated in two steps:
    1. The height is read from the element's containing block, which usually is the parent element. But what height is read? If the element has a position: static (the default) or a position: relative, then the height is read from the content-box of the containing block, but if the element has a position: absolute, then the height will be read from the padding-box of the containing block.
    2. Now the above read height is applied to the element according to the element's box-sizing property (box-sizing is not inherited).

Full height with flexbox

When using min-height (or max-height) instead of height for a parent, we aren't giving the element a fixed size, this means that percentage heights do not work for the children. Fortunately we have flexbox to help us expand the children to the full height.
