• Absolutely positioned elements are removed entirely from the document flow.
  • For absolutely positioned elements the containing block is the nearest relative, absolute or fixed positioned ancestor.
  • Fixed positioning is really just a specialized form of absolute positioning; elements with fixed positioning are fixed relative to the viewport/browser window rather than the containing element.
  • For a relatively positioned element the top, right, bottom and left properties specify the relative distance to shift the generated box. It's not meaningful to specify both top and bottom or left and right for the same element (one of the values will be ignored).
  • For absolute positioning the top, right, bottom and left properties can all be used at the same time. They specify the distance from each edge of the positioned element to the corresponding edge of the containing block.
  • The default value for the top, right, bottom and left properties is auto, which means the absolutely positioned box will appear exactly where it would have if it wasn't positioned.
