Margin concepts

  • Margins are meant to increase the distance between siblings, not to increase the gap between a child and its parent's bounding box; that's what padding is for.
  • Horizontal margins never change/collapse.
  • Vertical margins do not change/collapse:
    • For elements inside a flexbox.
    • For elements inside a css grid layout.
    • For floated elements
    • For absolute positioned elements.
    • For inline-block elements.
    • Across containers with overflow-x or overflow-y set to anything other than visible (like hidden, auto, or scroll).
  • For all the other cases, vertical margins change/collapse when they touch. A zero size margin can also touch another margin.
    Hint: when verifying whether margins touch, treat negative margins like positive ones.

Vertical margins calculation

  1. When all margins are positive, they collapse to the largest margin.
  2. When there are negative and positive margins, the size of the collapsed margin is the sum of the largest positive margin and the smallest (most negative) negative margin.
  3. When all margins are negative, they collapse to the smallest (most negative) margin.

margin: 8px 0

margin: 24px 0

margin: -4px 0

margin: 24px 0

margin: -4px 0

margin: -4px 0

Nested elements with touching vertical margins

When vertical margins touch between nested elements, the vertical margins calculation outlined above happens between them and the resulting margin is moved up to the parent:

margin: 24px 0

margin: 24px 0

margin: 24px 0

Calculation happens between all involved margins:

margin: 8px 0

margin: 24px 0

A padding, but also a border, can prevent the vertical margins from touching:

margin: 24px 0

margin: 24px 0

margin: 8px 0

margin: 24px 0
